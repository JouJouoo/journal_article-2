# IEEE 33/69 标准算例重跑实验协议（2026-06-05）

## 研究问题

在 IEEE 33-bus 与 IEEE 69-bus 标准配电网数据已切换为 pandapower/MATPOWER 权威来源并锁定为 `.npz` profile 后，复核 TECSF 及对比方法在标准算例上的可行性、结算稳定性、系统成本和碳排表现是否仍满足论文实验要求。

## 数据来源与标准性

使用官方/权威标准算例。IEEE 33-bus 由 `pandapower.networks.case33bw()` 加载并执行 `runpp` 校验；IEEE 69-bus 使用本地 `data/case69.m`（MATPOWER `case69`）解析。训练与评估阶段不动态依赖外部库，而使用由权威来源生成并校验通过的冻结 benchmark profile。

## 对比方法

本次仅重跑 IEEE 33/69 benchmark 部分，沿用项目 benchmark suite 中的 TECSF、MAPPO、constrained MAPPO、heuristic、greedy feasible、myopic optimal 等对比设置；不扩展新的算法或超参数搜索。

## 随机种子、训练轮数与评价轮数

- `smoke test`：seeds = `7, 42`，训练 `3 episodes`，评价 `2 episodes`。
- `100 episode` 短训：seeds = `7, 42`，训练 `100 episodes`，评价 `5 episodes`。
- `1000 episode` 正式实验：仅在前两档通过后执行；seeds = `7, 42, 100, 2026, 3407`，训练 `1000 episodes`，评价 `20 episodes`。

## 标准算例检查项

正式训练前必须确认：

- 动作尺度：动态动作边界开启，买/卖/储能动作不超过标准算例本地负荷尺度。
- 负荷尺度：IEEE 33 总负荷为 `3.7150 MW / 2.3000 MVAr`；IEEE 69 总负荷为 `3.8021 MW / 2.6947 MVAr`。
- 线路容量：保留 TECSF benchmark 线路容量约束，不采用 pandapower 的占位型超大热限。
- 储能边界：`soc_min/soc_max`、充放电上限和安全投影开启。
- 可行性投影：动态 action bounds、safety shield、emergency balancing 开启。
- 指标口径：系统社会成本、参与者支付成本、购电碳排、结算成功率、最大约束违背、problem count 口径与现有报告一致。

## 核心指标与验收阈值

- source validation：`scripts/validate_benchmark_sources.py --case69-m data\case69.m --strict` 必须通过。
- smoke：两套标准算例均完成训练、评价、统计和图表后处理，无异常退出、无 NaN。
- 100 episode 短训：两套标准算例 summary 生成完整，`problem_count = 0`，结算成功率接近或达到 `1.0`，最大约束违背不出现系统性越限。
- 1000 episode 正式实验：通过 `scripts/check_experiment_acceptance.py` 或等价验收；关键表格和图件可用于论文更新。

## 允许支持的论文结论

`smoke test` 只支持“代码路径和标准算例 profile 可运行”。`100 episode` 短训只支持“修正后的标准算例设置没有明显可行性回归”。只有 `1000 episode` 正式实验通过后，才允许更新论文中关于 IEEE 33/69 外部有效性、成本、碳排和结算稳定性的正式结论。
