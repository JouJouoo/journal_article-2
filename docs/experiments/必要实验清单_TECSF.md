# 论文必要实验清单

> 生成日期：2026-07-05
> 依据论文：面向 P2P 低碳能源交易的低碳资产感知的 MAPPO 算法
> 依据代码：当前工作区最新代码

## 1. 实验定位

论文的核心贡献有三条，实验需围绕它们展开，不追求"大而全"的工程验证：

1. **LCCoins 生成机制**：锚定用户低碳贡献的区块链代币，通过联盟链共识验证后铸造为可累计的链上低碳资产。
2. **低碳资产优势分解 MAPPO**：将优势估计分解为经济优势和 LCCoins 低碳资产优势，引入信用分配 critic 和自适应 PPO 裁剪。
3. **基于 LCCoins 动态资产效用的奖励函数**：CRRA 存量效用 + 增量效用，使策略兼顾当前低碳收益与长期资产积累。

## 2. 论文主张与实验对应关系

| 论文主张 | 必须提供的证据 | 对应实验 |
|----------|---------------|---------|
| 低碳资产感知 MAPPO 在经济性和低碳性上优于基线 | 与基线在相同场景、训练预算、种子下的统计比较 | E1 主对比实验 |
| 联盟链共识结算实现可审计、可回滚、防重复的电碳协同结算 | 哈希校验、失败注入、重复铸造、余额回滚、逐节点确认的压力测试 | E2 可信结算实验 |
| LCCoins 激励通过已确认结算结果引导低碳行为 | 去掉 LCCoins、去掉资产反馈、使用预设低碳奖励后的行为差异 | E3 LCCoins 激励实验 |
| 低碳资产优势分解和自适应裁剪优于标准 MAPPO | 与标准 MAPPO、无信用分配、固定裁剪的收敛曲线和测试表现比较 | E4 算法消融实验 |
| 拉格朗日约束和结算拒绝机制控制网络安全 | 与无拉格朗日/固定惩罚在紧约束场景下的违约次数比较 | E5 约束安全实验 |
| 方法可迁移到标准配电网 profile | IEEE 33/69 算例上的迁移表现 | E6 标准算例验证 |

## 3. P0 必须完成的实验

### E1. 主对比实验

**目的**：证明低碳资产感知 MAPPO 相比基线在经济性、低碳性和学习收敛方面有稳定改进。

**对比方法**：
- `tecsf`（论文主方法，等价于 `lc_mappo`）
- `mappo`：标准 MAPPO，无 LCCoins 资产观测
- `no_chain`：绕过联盟链原子结算
- `no_lccoins`：保留结算但不铸造 LCCoins
- `no_lagrange`：去除动态拉格朗日惩罚
- `preset_low_carbon`：模型内生低碳奖励，不经联盟链确认
- `heuristic`：确定性启发式基线

**设置**：
- 训练不少于 1000 episode，不少于 5 个种子（`{7, 42, 100, 2026, 3407}`）
- 训练完成后固定策略，在 20-50 个测试场景上评估

**必须报告的指标**：
- 经济性：系统总运行成本、P2P 成交电量、主体平均收益
- 低碳性：系统碳排放、净碳配额需求、可再生能源消纳率
- 安全性：结算成功率、最大约束违背
- 学习性：末 100 episode 平均回报、收敛速度

### E2. 可信结算实验

**目的**：证明联盟链共识结算不只是记录交易，而是实现了可审计、可回滚、防重复的电碳协同结算。

**实验设计**：
1. 正常结算：合法出清包，记录 `Pending → Verified → Settled`，验证逐节点确认标识 σ^v
2. 哈希篡改：修改出清包内容或哈希 → `Rejected`
3. 约束违背：构造超阈值出清包 → `Rejected`
4. 执行失败：注入余额不足或强制异常 → `Reverted`，验证三类余额完全回滚
5. 重复铸造：同一记录、主体、时段重复铸造 → 被阻止
6. 审计一致性：链上记录哈希、出清包、回执、状态根一致复核

**必须报告的指标**：
- 结算成功率、拒绝率、回滚率
- 错误检出率（篡改、重复铸造、约束越限是否 100% 识别）
- 回滚后余额误差（应为 0 或数值容差内）
- 验证节点确认分布（全票/零票模式、阈值通过情况）

### E3. LCCoins 激励实验

**目的**：证明 LCCoins 不是简单增加奖励项，而是通过已确认结算结果引导低碳行为。

**对比方法**：
- `tecsf`（完整版）
- `no_lccoins`：保留结算但不铸造 LCCoins
- `preset_low_carbon`：模型内生低碳奖励，不经联盟链确认

**必须报告的指标**：
- LCCoins 总量和按主体分布
- 光伏消纳量、低碳贡献抵扣量、剩余低碳贡献售卖量
- 电网购电碳排放、净碳配额需求
- LCCoins 与实际低碳贡献的相关性
- 是否存在奖励重复（已售卖的低碳贡献不应再次获得 LCCoins）

**敏感性分析**：
- 扫描 LCCoins 效用权重（`stock_utility_weight`、`increment_utility_weight`）
- 扫描铸造系数 `minting_coefficient`

### E4. 算法消融实验

**目的**：证明低碳资产优势分解、信用分配和自适应裁剪各自有独立贡献。

**对比设置**：
- `tecsf` vs `mappo`：标准 MAPPO 无资产观测、无循环网络
- `tecsf` vs `no_lccoins`：去掉 LCCoins 资产观测和效用
- `tecsf` vs `no_lagrange`：去掉动态约束乘子

**必须报告的指标**：
- 训练曲线（多 seed 均值）
- 末 100 episode 平均回报
- 收敛速度
- 测试集表现
- 策略稳定性

### E5. 约束安全实验

**目的**：证明拉格朗日约束和结算拒绝机制能有效控制网络安全和碳约束违背。

**实验设计**：
- 增加紧约束场景：降低线路容量、增大负荷/光伏波动
- 比较 `tecsf`、`no_lagrange`、`heuristic`

**必须报告的指标**：
- 电压/线路/SOC 越限次数和幅度
- 交易修正量
- 结算拒绝/回滚次数
- 安全性提升是否以过大经济损失为代价

## 4. P1 强烈建议完成的实验

### E6. 标准算例验证

**目的**：证明方法可迁移到标准配电网 profile，但不主张成本和碳排全面最优。

**实验设计**：
- IEEE 33-bus 和 IEEE 69-bus 算例
- 使用 pandapower/MATPOWER 作为权威来源，冻结 `.npz` profile 用于训练和评估
- `synthetic33` 仅作为派生 profile，不报告为标准算例

**必须报告的指标**：
- 结算成功率、最大约束违背
- 系统成本、碳排放
- 与合成场景结果的对比

### E7. 鲁棒性与参数敏感性

**建议扫描维度**：
- 光伏渗透率：低/中/高
- 碳价：低/中/高
- 电网排放因子：固定/变化
- 产消者规模：8/16/32，节点规模：5/33/69

### E8. 可扩展性和计算开销

**建议指标**：
- 单时段出清时间、单笔结算确认时间
- 每 episode 总运行时间
- 账本存储开销随区块数变化

## 5. 最低可投稿实验包

如果时间有限，最低限度应完成：

1. **E1 主对比**：7 方法 × 5 种子 × 1000 episode + 20 测试场景评估
2. **E2 可信结算**：6 类压力测试场景，输出可展示表格
3. **E3 LCCoins 激励**：3 方法对比 + 参数敏感性
4. **E4 算法消融**：训练曲线 + 测试表现
5. **E5 约束安全**：紧约束场景对比
6. **E6 标准算例**：IEEE 33/69 迁移验证

## 6. 注意事项

- 论文不主张经济性/碳排放全面最优，如实报告弱项
- `synthetic33` 只能表述为派生/合成 profile
- 联盟链为本地可审计结算模拟器，不展开链上吞吐量、确认时延等工程性能
- LCCoins 权重需要控制，过大可能导致回报塌陷（已有 `kappa=0.1` 诊断）
