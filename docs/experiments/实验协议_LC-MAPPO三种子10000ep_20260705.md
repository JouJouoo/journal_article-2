# 实验协议：LC-MAPPO 三种子 10000 episodes 稳定性与鲁棒性评估

**日期**: 2026-07-05  
**实验目的**: 在相同配置下使用三个不同随机种子运行 LC-MAPPO 完整方法，评估算法在多随机种子下的训练稳定性和性能鲁棒性。

## 1. 研究问题

- LC-MAPPO 方法在不同随机种子下的训练收敛性是否稳定？
- 不同随机种子下的最终性能指标（奖励、P2P 交易量、碳排放、LCCoins）是否有显著波动？
- 三种子的均值与标准差能否支撑论文中关于算法鲁棒性的结论？

## 2. 实验配置

### 2.1 方法变体
- **variant**: `lc_mappo`（完整 LC-MAPPO 方法）
- **特性**: 全部组件开启（区块链共识 + LCCoins + 循环网络 + 拉格朗日安全约束 + 资产感知 + 双重优势分解）

### 2.2 随机种子
| 种子编号 | 用途 | 状态 |
|---------|------|------|
| 7 | 已有结果 | 已完成 |
| 42 | 新增重复实验 | 待运行 |
| 100 | 新增重复实验 | 待运行 |

种子选择来源：`configs/experiment_protocol.yaml` 的 calibration seeds `[7, 42, 100]`。

### 2.3 训练参数（全部与种子 7 保持一致）
- **episodes**: 10000
- **config**: `configs/default.yaml`
- **device**: cpu（与种子 7 一致，保守估计）

关键超参数（从 `configs/default.yaml`）：
- `hidden_dim`: 64
- `recurrent_dim`: 64
- `actor_lr`: 5e-5
- `critic_lr`: 1e-3
- `gamma`: 0.98
- `clip_eps`: 0.2
- `entropy_coef`: 0.01
- `horizon`: 24
- `num_agents`: 8

### 2.4 环境配置
- 使用合成 profile（与种子 7 一致）
- `scenario.profile_path: ""`（默认合成数据）

## 3. 对比方法

无需对比方法，本实验仅评估 LC-MAPPO 本身在不同种子下的稳定性。

### 4.2 评估指标
主指标（来自 `experiment_protocol.yaml`）：
- `mean_reward` — 每个 episode 的平均奖励
- `feasible_rate` — 可行性比例
- `settlement_success_rate` — 结算成功率
- `max_violation` — 最大约束违反

辅助指标：
- `system_cost` — 系统总成本
- `grid_carbon_emission` — 电网碳排放
- `p2p_energy` — P2P 交易量
- `lccoins` — LCCoins 铸造量
- `carbon_reduction` — 碳减排量

### 4.3 统计方法
- 每个种子的最终 1000 episode 统计（均值 ± 标准差）
- 三种子间比较：均值 ± 标准差
- 收敛曲线叠加图（含均值 ± 标准差阴影）

## 5. 验收标准

- 两个新增种子均完成 10000 episodes 训练，无报错
- 三种子训练曲线叠加图生成成功
- 三种子间最终 1000-episode 奖励标准差可接受（建议 < 最终均值的 20%）
- 所有种子的 `settlement_success_rate` >= 0.95
- 所有种子的 `feasible_rate` >= 0.95

## 9. 风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| 训练时间过长被中断 | 使用后台执行，中间 checkpoint 可恢复 |
| 内存不足 | 单进程顺序执行，不并行 |
| 种子 42/100 结果与种子 7 偏差过大 | 报告方差，分析原因（可能是超参数需要再调优） |
| 脚本路径或环境问题 | 先执行 smoke test（5 episodes）验证脚本可用性 |
