## 1. 实验定位

当前论文实验不应继续扩成“大而全”的工程验证。TECSF 论文的核心不是证明方法在所有成本、碳排、规模和真实扰动场景下全面最优，而是证明以下四点：

1. TECSF 能形成可审计的电碳协同结算闭环。
2. LCCoins 和结算反馈确实参与策略学习。
3. 拉格朗日约束和结算拒绝机制能提高安全可行性。
4. 方法可以迁移到 IEEE 33/69 标准配电网 profile，但不声称成本和碳排全面最优。

因此，投稿版实验采用“最小充分实验包”：保留能支撑论文核心贡献的实验，删除或降级高成本但不直接支撑主张的大规模扩展、真实全年时间序列、多优化器基线和复杂攻击模型。

## 2. 当前结果的关键诊断

| 发现 | 已有证据 | 论文含义 |
| --- | --- | --- |
| 可信结算证据强 | 主实验 TECSF 结算成功率 1.000，最大约束违背 0.000；结算压力 30/30 通过 | 可以支撑可信结算、审计、回滚主张 |
| 成本和碳排不全面最优 | 主实验 TECSF 系统成本高于 MAPPO；IEEE 33/69 中 TECSF 不支配 myopic_opt | 不能写成经济性/碳排全面领先 |
| 约束机制必要 | no_lagrange 主实验结算成功率 0.236，最大约束违背 9.360 | 拉格朗日约束和结算拒绝是核心模块 |
| LCCoins 权重需要控制 | `kappa=0.1` 时 TECSF 平均回报塌陷、结算成功率下降 | 敏感性实验必须保留，但不需要扩大为大网格 |
| 大规模场景有边界 | `16 agents / 9 nodes` 中 TECSF 结算成功率 0.480，最大约束违背 5.049 | 可扩展性只能作为边界分析，不主张强扩展性 |
| IEEE 33/69 是外部验证 | 标准算例已跑 30 个 1000-episode run | 可证明机制迁移，但要如实报告弱项 |

## 4. 各实验的具体设置

### E1. 主对比与核心消融

建议方法：

```text
tecsf
mappo
constrained_mappo
myopic_opt
heuristic
no_lccoins
no_feedback
no_lagrange
```

其中 `constrained_mappo` 用于补充约束 RL 基线，`myopic_opt` 用于提供轻量优化型确定性基线。`safety_only` 可以放入补充材料，不必进入主表。

必须报告指标：

```text
system_social_cost
participant_payment_cost
grid_carbon_emission
net_carbon_allowance_need
lccoins
mean_reward
settlement_success_rate
feasible_rate
max_violation
runtime
```

解释口径：TECSF 的优势主要看可信结算、约束安全、低碳反馈闭环和可审计性；若成本或碳排不是最低，应如实说明。

### E2. LCCoins 敏感性

只保留 `kappa={0,0.1,0.2,0.5}`。主文不需要展示 `alpha_q`、`alpha_offset`、`violation_penalty` 的联合大网格。

需要回答的问题：

1. LCCoins 奖励是否影响回报和行为。
2. 奖励权重是否会影响安全可行性。
3. 默认 `kappa=0.2` 的选择是否有结果依据。

若空间有限，主文只放 TECSF 在不同 `kappa` 下的表格，no_lccoins 作为参照线即可。

### E3. 网络约束压力

只保留两个代表场景：

```text
line=0.7, trade=1.0
line=0.5, trade=1.0
```

方法建议：

```text
tecsf
no_lagrange
heuristic 或 myopic_opt
```

该实验只需证明：去掉约束机制会显著增加越限和结算失败；TECSF 能以一定成本代价提升安全性。不要把压力实验扩大成大量 line/trade/load/PV/carbon price 组合。

### E4. TECS-Chain 结算压力与 ledger

保留现有 6 类 case：

```text
正常结算
哈希篡改
约束越限
执行失败回滚
重复铸造
审计一致性
```

报告重点：

1. 总 case 数和通过数。
2. 篡改、越限、重复铸造是否被拒绝。
3. 执行失败后能量余额、碳余额、LCCoins 余额最大回滚误差是否为 0。
4. ledger 图展示 block、receipt、state root、余额和 LCCoins 记录。

这部分不需要扩大到复杂攻击模型、共识机制或真实链部署。

### E5. IEEE 33/69 标准算例

保留 IEEE 33-bus 和 IEEE 69-bus。方法只需：

```text
tecsf
myopic_opt
heuristic
```

该实验的作用是外部有效性和边界说明，而不是证明 TECSF 支配优化基线。论文应写成：

```text
TECSF 的可信结算和 LCCoins 反馈链路能够迁移到标准配电网 profile；
但在成本和碳排上不全面优于 myopic_opt。
```

需要特别区分 `ieee33bw`、`ieee69` 与此前的 synthetic profile。synthetic profile 不能替代标准算例结论。

### E6. 可扩展性边界

可扩展性实验只作为局限性分析。保留 8/16 agents 与 5/9 nodes 即可，不建议主文扩展到 32 agents、17 nodes 或 33 nodes。

报告重点：

1. 运行时间随规模增加的趋势。
2. `16 agents / 9 nodes` 等失败或弱项场景。
3. 当前实现需要进一步改进可行性投影和训练效率。

## 5. 可以降级或删除的复杂项

| 项目 | 建议 | 原因 |
| --- | --- | --- |
| 32 agents / 17 或 33 nodes | 不作为主文必需 | 当前 16 agents / 9 nodes 已暴露边界，继续扩展会拉长工作量 |
| 多天气、多季节、真实全年时间序列 | 后续工作 | 对外部有效性有帮助，但不是本文机制证明的必要条件 |
| 完整 MILP/MIQP/AC-OPF 基线 | 可选补充 | 实现成本高，且会把论文重心转向调度最优性 |
| 大量 carbon price、PV penetration、storage capacity 网格 | 附录或后续 | 容易稀释主线，当前 LCCoins 与网络压力已足够说明敏感性 |
| 复杂攻击模型、隐私保护、共识机制对比 | 不做 | 当前 TECS-Chain 是本地可审计状态机，不主张真实链部署或隐私协议 |
| 过多统计方法 | 保留最小统计 | 95% CI、配对检验、Holm 校正即可 |

## 6. 统计与表述规则

主文至少采用 5 个随机种子，并报告均值和标准差。对于“优于”“显著改善”等表述，需要有配对统计或置信区间支持。

建议规则：

1. 主指标：`settlement_success_rate`、`feasible_rate`、`max_violation`、`mean_reward`。
2. 次指标：`system_social_cost`、`participant_payment_cost`、`grid_carbon_emission`、`net_carbon_allowance_need`、`lccoins`、`runtime`。
3. 安全门槛：若 `settlement_success_rate < 0.95` 或 `max_violation > 0.1`，则标记为 infeasible，不参与成本/碳排最优排序。
4. 多方法比较使用配对检验；多指标比较使用 Holm 校正即可。
5. 未通过统计支持的指标只写“接近”“相当”“未观察到显著优势”，不要写“显著优于”。

## 7. 最小验收标准

最终论文实验只需满足以下门槛：

1. 主对比中 TECSF 的 `settlement_success_rate >= 0.95`，`max_violation <= 0.1`。
2. no_lagrange 在主实验或网络压力中表现出明显安全退化。
3. no_lccoins 或 no_feedback 相比 TECSF 在回报、LCCoins 或反馈完整性上有可解释差异。
4. TECS-Chain 压力测试全部通过，回滚误差为 0。
5. IEEE 33/69 至少证明 TECSF 可运行、可结算、可审计，并如实报告弱项。
6. 成本和碳排不全面占优时，结论主动收缩到“可信结算、安全可行和低碳反馈闭环”。

## 8. 推荐执行顺序

1. 先用当前最新代码跑 quick suite，确认脚本和指标没有断裂。
2. 重跑 E1 主对比与消融，生成主表。
3. 重跑 E5 IEEE 33/69，确认动作投影和成本口径修正后的标准算例表现。
4. 只重跑压缩版 E2/E3，不跑完整大网格。
5. 复用或轻量重跑 E4 结算压力和 ledger 案例。
6. 生成统计检验和 feasible-gate 标记，用于约束论文表述。
7. 最后更新论文结论，避免出现“全面最优”式过度主张。

## 9. 论文结论口径

不建议写成：

```text
TECSF 在经济成本和碳排放上全面优于所有方法。
```

建议写成：

```text
TECSF 能在 P2P 电碳交易中提供可审计的低碳激励和可信结算闭环；
相比去掉 LCCoins 或拉格朗日约束的消融模型，TECSF 提高了结算安全性、约束可行性和低碳反馈完整性；
标准 IEEE 33/69 算例进一步表明该机制可迁移到公开配电网 profile，但成本和碳排优势并不全面成立。
```

这个口径与当前数据和最新代码更一致，也更符合学术论文对证据边界的要求。
