# 项目执行规则

本文件用于约束 AI 在本项目中的后续工作。任何实验、代码修改、论文图表、Word 文档处理或 Git 提交前，必须先遵守以下流程。

## 1. 实验前协议

新实验开始前，必须先生成 1 页实验协议，至少包含：

- 研究问题
- 使用数据是否为官方标准算例
- 对比方法
- 随机种子
- 训练轮数
- 评价轮数
- 核心指标
- 验收阈值
- 该实验允许支持的论文结论

未生成实验协议，不允许直接开始正式实验。

## 2. 实验执行顺序

实验必须按以下顺序推进：

1. `smoke test`
2. `100/200 episode` 短训
3. `1000 episode` 正式实验

除非用户明确要求，否则不要直接运行全量实验。

## 3. 标准算例检查

使用 IEEE 33/69 或其他标准算例前，必须先检查：

- 动作尺度
- 负荷尺度
- 线路容量
- 储能边界
- 可行性投影
- 指标口径

检查未通过，不允许直接训练。

## 4. 图表和 Word 文档

论文图表必须先导出独立的 `png` 和/或 `pdf` 文件，并检查：

- 图例是否遮挡
- 坐标轴是否完整
- 字体是否清晰
- 线条、颜色、标注是否适合论文插入
- 插入 Word/docx 后是否出现遮挡、错位或无法读取内容

确认独立图表质量后，再插入 Word/docx。

## 5. Git 提交

Git 提交前必须执行：

- 查看 `git status`
- 检查 `.gitignore`
- 区分源码、论文文件和实验输出
- 只提交用户要求的文件范围
- 实验输出单独归档，不默认提交

禁止在未确认提交范围时把全部生成文件一起提交。

## 6. AI 响应要求

AI 在执行项目任务前，必须先说明当前属于哪一类任务，并给出对应检查项是否已完成。若检查项缺失，必须先补齐检查项，再进入正式执行。
